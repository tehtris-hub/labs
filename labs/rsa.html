La Clé Secrète du Patron (GPG)

Vous avez changé d'ordinateur. Coup de chance, vous avez sauvegardé votre clé privée GPG (backup_key.gpg). Votre patron a envoyé un nouveau message chiffré (message.txt.gpg). Sans votre clé privée, c'est illisible. Mais vous l'avez. Alors au travail !

Concept : RSA et GPG

Concept clé
La boîte aux lettres magique : clé publique vs clé privée
Imaginez une boîte aux lettres spéciale : la fente (clé publique) — tout le monde peut y glisser un message. La clé (clé privée) — seul vous pouvez l'ouvrir. Votre patron a utilisé VOTRE fente. Seule VOTRE clé privée peut l'ouvrir. La clé publique ? Collez-la partout — aucun risque. La clé privée ne quitte jamais votre trousseau.

GPG (GNU Privacy Guard) est l'implémentation libre du standard OpenPGP. Il utilise RSA pour chiffrer et signer des messages et des fichiers.

02 //

Importer la clé de sauvegarde

Vous avez récupéré votre backup_key.gpg depuis votre ancien ordinateur. Il faut l'importer dans votre trousseau GPG.

import_key.sh
# Voir les clés GPG actuellement disponibles
gpg --list-keys
gpg --list-secret-keys

# TODO: Importer la clé privée depuis backup_key.gpg
# Syntaxe : gpg --import FICHIER
gpg --?????? backup_key.gpg

# Vérifier que la clé a bien été importée
gpg --list-secret-keys
▸ AFFICHER LA SOLUTION (essayez d'abord !)

L'option pour importer une clé GPG est tout simplement --import :

gpg --import backup_key.gpg
03 //

Déchiffrer le message

Votre clé privée est importée. GPG peut maintenant déchiffrer automatiquement les messages qui vous sont destinés.

decrypt_message.sh
# TODO: Déchiffrer le message du patron
# Syntaxe : gpg --decrypt FICHIER_CHIFFRE
gpg --?????? message.txt.gpg

# Alternative avec redirection vers un fichier
gpg --output message_clair.txt --decrypt message.txt.gpg

# Lire le message
cat message_clair.txt
▸ AFFICHER LA SOLUTION (essayez d'abord !)

L'option est --decrypt (ou -d en version courte) :

gpg --decrypt message.txt.gpg
# ou pour sauvegarder dans un fichier :
gpg --output message_clair.txt --decrypt message.txt.gpg
04 //

Vérifier et nettoyer

Confirmez que le message est bien lisible et examinez les métadonnées de votre clé.

Vérification finale
# Confirmer que le message est bien lisible
cat message_clair.txt

# Voir les métadonnées de la clé
gpg --list-keys --with-fingerprint

# Bonus : exporter votre clé publique (à partager librement)
gpg --export --armor votre@email.com > ma_cle_publique.asc
MESSAGE DÉCHIFFRÉ ! Vous avez déchiffré le message du patron ! Spoiler : c'était probablement une réunion à 8h du matin. Au moins, personne d'autre ne l'a lu.
