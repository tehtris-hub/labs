Survivre dans un Terminal

C'est quoi un terminal ?

Concept clé
Le terminal, c'est comme envoyer des SMS à votre ordinateur
Cliquer sur une icône, c'est demander à un assistant de faire quelque chose pour vous. Le terminal, c'est parler directement au patron. Plus puissant, plus rapide — et oui, ça fait plus geek en réunion.

Un terminal est une interface texte pour communiquer avec votre système. À l'intérieur tourne un programme appelé le shell — sur Linux, c'est généralement bash (Bourne Again SHell).

Le prompt user@hostname:~$ signifie : connecté en tant que user, sur la machine hostname, dans ~ (votre home), droits normaux ($).

Premiers pas dans le terminal
# Qui suis-je ?
whoami

# Sur quelle machine je suis ?
hostname

# Quelle version de bash ?
bash --version

# Afficher un message (premier programme !)
echo "Bonjour le terminal !"
02 //

Où suis-je ?

pwd (print working directory) c'est votre GPS : il vous dit exactement où vous vous trouvez. ls (list) c'est l'inventaire de la pièce dans laquelle vous êtes.

Les colonnes de ls -la permissions (drwxr-xr-x) · liens · propriétaire · groupe · taille · date · nom. Le d = répertoire, - = fichier, l = lien symbolique.
Se repérer dans le système de fichiers
# Ma position actuelle
pwd

# Voir ce qu'il y a ici
ls

# Version détaillée (permissions, taille, date)
ls -la

# Voir seulement les .txt
ls *.txt
03 //

Me déplacer

cd c'est comme ouvrir un dossier — mais 10x plus rapide. cd ~ c'est le bouton Maison. cd - c'est le bouton précédent de votre navigateur.

Navigation dans les répertoires
# Aller dans /tmp
cd /tmp

# Voir où on est
pwd

# Remonter d'un niveau
cd ..

# Retour à la maison (home directory)
cd ~

# Retour au répertoire précédent (super pratique !)
cd -

# Créer un nouveau dossier
mkdir mon_dossier

Lire des fichiers

Le patron vous a envoyé un fichier de notes. cat affiche tout. less lit page par page. tail -f suit un fichier en temps réel — idéal pour les logs.

Lire et suivre des fichiers
# Tout afficher d'un coup
cat /etc/hostname

# Lire page par page (q pour quitter)
less /etc/passwd

# Les 5 premières lignes seulement
head -5 /var/log/syslog

# Les 5 dernières lignes
tail -5 /var/log/syslog

# Suivre un fichier en temps réel (comme watch -n1 mais mieux)
tail -f /var/log/syslog
05 //

Chercher

Analogie
grep = Ctrl+F du terminal. find = chien renifleur.
grep cherche du texte dans les fichiers. find cherche des fichiers par leur nom, taille, date... Les deux ensemble sont imbattables.
Recherche dans le système
# Chercher 'root' dans un fichier
grep "root" /etc/passwd

# Chercher sans tenir compte de la casse
grep -i "error" /var/log/syslog

# Chercher récursivement dans un dossier
grep -r "password" /etc/

# Trouver tous les fichiers .log dans /var
find /var -name "*.log"

# Trouver les fichiers modifiés il y a moins de 24h
find /tmp -mtime -1
06 //

Tuer des processus

Analogie
ps aux = liste des employés. kill = dire "rentre chez toi". kill -9 = moins diplomatique.
Chaque programme a un PID. kill envoie un signal arrête-toi poliment. kill -9 coupe l'alimentation — pas de nettoyage, juste fin.
Gestion des processus
# Voir tous les processus
ps aux

# Trouver un processus par nom
ps aux | grep ssh

# Tuer proprement un processus (remplacez PID par l'ID)
kill 1234

# Tuer sans pitié (si le processus est têtu)
kill -9 1234

# Vue interactive (q pour quitter)
top
VOUS Y ÊTES PRESQUE Vous savez maintenant naviguer, lire des fichiers, chercher et gérer des processus. Votre souris vous regarde avec des yeux tristes. C'est normal.
