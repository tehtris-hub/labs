Le Trafic Suspect de la Nuit

L'équipe sécurité vient de vous alerter : quelque chose envoie des requêtes HTTP vers 10.0.0.10:8080 toutes les 30 secondes. Qui fait ça ? Quand ? Pourquoi ? C'est votre job de le découvrir — et d'arrêter ça.

Concept : Les Ports et Connexions

Concept clé
Votre ordinateur a 65535 portes — et il faut savoir lesquelles sont ouvertes
Chaque service utilise un 'port' — une porte numérotée. HTTP = porte 80. HTTPS = 443. SSH = 22. Quand un programme fait une connexion sortante : 'sortie par la porte 54321 vers le voisin port 8080'. ss liste toutes ces connexions en cours.
02 //

03 //

Identifier et tuer le processus

Vous avez trouvé la connexion suspecte. Maintenant, identifiez quel processus en est responsable et terminez-le.

Identification et suppression du processus
# Trouver le PID du processus qui fait les requêtes
ss -anp | grep 8080
# ou
lsof -i :8080

# Voir les détails du processus (remplacer PID par l'ID trouvé)
ps aux | grep PID

# Tuer le processus proprement
kill PID

# Si résistant :
kill -9 PID

# Vérifier que la connexion a disparu
ss -anp | grep 8080
04 //

Vérification finale

Confirmez que le trafic est stoppé et cherchez si quelque chose relaierait ce processus au démarrage.

Vérification et recherche de persistance
# Confirmer qu'aucune connexion vers 8080 n'est active
ss -anp | grep 8080
# → doit être vide

# Chercher le script qui lançait ça au démarrage
grep -r "10.0.0.10" /etc/cron* /etc/rc* ~/.bashrc 2>/dev/null
TRAFIC STOPPÉ ! Le trafic suspect est stoppé. L'équipe sécurité vous dit merci. Cédric dit que c'est pas lui.
