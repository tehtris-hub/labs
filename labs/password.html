Cédric et le Mot de Passe en Carton

Cédric, votre collègue adoré, a configuré son compte avec le mot de passe 'password'. Oui. 'password'. Comme si on mettait un verrou en plastique sur la porte du coffre-fort. Votre mission : sauver l'honneur de l'équipe.

Concept : Les Mots de Passe

Concept clé
Un mot de passe, c'est le verrou de votre maison numérique
'password' c'est une porte grande ouverte. '16 caractères avec @, chiffres, majuscules' c'est un coffre-fort. Les hackers ont des dictionnaires de milliards de mots de passe courants — 'password' est dans toutes les listes. Cédric était littéralement dans un dictionnaire.

Une attaque par force brute essaie toutes les combinaisons (comme craquer un cadenas à 4 chiffres en testant 0000, 0001...). Une attaque par dictionnaire est plus maligne : elle essaie d'abord les plus courants. Cédric était en première page.

02 //

Diagnostiquer

Avant de corriger, vérifions que le compte de Cédric existe et regardons son état.

Inspection du compte cedric
# Vérifier que le compte cedric existe
id cedric

# Voir les infos du compte
cat /etc/passwd | grep cedric

# Voir quand le mot de passe a été changé pour la dernière fois
sudo chage -l cedric
CE QUE VOUS ALLEZ VOIR chage -l affiche la date du dernier changement, la date d'expiration. Si la date est ancienne... Cédric n'a JAMAIS changé son mot de passe. Quelle surprise.
03 //

Corriger le script de Cédric

Cédric a écrit un script pour gérer les comptes, mais il a remplacé les vraies options par des ?. À vous de compléter les trous !

fix_cedric.sh
#!/bin/bash
# Cédric a écrit ce script mais il ne fonctionne pas...
# (il a remplacé les vraies options par des ?)

# Étape 1 : Bloquer le compte pendant qu'on travaille
sudo usermod -? cedric
# (Indice : quelle option de usermod BLOQUE un compte ?)

# Étape 2 : Définir un nouveau mot de passe fort
# Nouveau mot de passe : fait_1_3ffort,C3dric
sudo passwd c?????
# (Indice : quel est le nom d'utilisateur ?)

# Étape 3 : Débloquer le compte
sudo usermod -? cedric
# (Indice : quelle option DÉBLOQUE ?)
▸ AFFICHER LA SOLUTION (essayez d'abord !)

Voici le script corrigé. -L = Lock (bloquer). -U = Unlock (débloquer).

# Bloquer le compte
sudo usermod -L cedric

# Changer le mot de passe
sudo passwd cedric
# Entrez : fait_1_3ffort,C3dric

# Débloquer le compte
sudo usermod -U cedric
04 //

Vérifier

Vérifions que le nouveau mot de passe est en place et que l'ancien ne fonctionne plus.

Vérification finale
# Vérifier que le compte est débloqué
sudo passwd -S cedric

# Tenter une connexion avec l'ancien mot de passe (doit échouer)
su - cedric
# Entrez 'password' → FAIL (comme prévu)

# Confirmer avec le nouveau mot de passe
# Entrez 'fait_1_3ffort,C3dric' → SUCCESS
MISSION ACCOMPLIE Le compte de Cédric est sécurisé. Il n'est pas content, mais c'est pour son bien. Et pour le bien de toute l'équipe.
